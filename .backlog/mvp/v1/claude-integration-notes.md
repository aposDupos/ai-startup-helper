# Заметки по интеграции обратной связи

**Дата:** 2026-02-28

---

## Интегрированные предложения

### ✅ 1. GigaChat PoC в Фазу 1
**Предложение:** Добавить GigaChat API PoC как blocking task в Phase 1.  
**Решение:** Интегрировано. Добавлена задача «GigaChat API PoC» в Фазу 1 (auth, chat, streaming, embeddings). Это критический путь.

### ✅ 2. Ранний RAG Prototype
**Предложение:** Начать RAG prototype в Фазе 2. 
**Решение:** Интегрировано. В Фазу 2 добавлена задача минимального RAG прототипа (загрузка 2–3 документов, embedding, similarity search).

### ✅ 3. Согласие родителей — UX flow
**Предложение:** Описать flow получения согласия для несовершеннолетних.  
**Решение:** Интегрировано. Добавлено в Фазу 1 onboarding: при возрасте < 18 → email-подтверждение от родителя.

### ✅ 4. Параллельное создание контента
**Предложение:** Начать контент Academy раньше.  
**Решение:** Интегрировано. Контент-работа начинается с Фазы 1 параллельно с разработкой.

### ✅ 5. Кэширование AI-ответов
**Предложение:** Добавить стратегию кэширования.  
**Решение:** Интегрировано. Кэширование RAG-результатов и embedding-запросов через Redis/in-memory.

### ✅ 6. AI Observability
**Предложение:** Добавить мониторинг AI (latency, tokens, errors).  
**Решение:** Интегрировано. Кастомный logging AI-вызовов в PostgreSQL (prompt, response, latency, tokens).

## Неинтегрированные предложения

### ❌ 7. AI Eval Set (50+ пар)
**Предложение:** Создать eval-набор для regression testing.  
**Причина отклонения:** Уместно, но 50+ пар — избыточно для MVP. Начнём с 10–15 ключевых сценариев. Масштабируем в v2.

### ❌ 8. SberID обязательность
**Предложение:** Уточнить, обязателен ли SberID.  
**Причина отклонения:** Вопрос для стейкхолдера. Пока реализуем email + VK OAuth, SberID как опция в плане.
